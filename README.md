# Personal Website

This repository is set up as a GitHub Pages personal portfolio and study-note site using Jekyll and the Minimal Mistakes theme.

You do not need to write HTML for normal updates. Edit Markdown files (`.md`) and the site will be generated by GitHub Pages.

## Main Files

- `_config.yml`: site title, author profile, email, GitHub, LinkedIn, and theme settings.
- `index.md`: homepage.
- `_pages/about.md`: about page.
- `_pages/research.md`: research and projects page.
- `_pages/cv.md`: CV page.
- `_pages/study.md`: study-note index.
- `_posts/`: study notes and blog-style posts.

## Customize Your Profile

Edit `_config.yml`:

```yaml
title: "Your Name"
name: "Your Name"
description: "Personal portfolio, research notes, and study records."
url: "https://your-github-username.github.io"
baseurl: ""
repository: "your-github-username/your-github-username.github.io"
```

Also update your email, GitHub, LinkedIn, and Google Scholar links in the `author` section.

To replace the profile image, add your photo under `assets/images/` and update:

```yaml
avatar: "/assets/images/your-photo.jpg"
```

## Add a Study Note

Create a file in `_posts` with this name pattern:

```text
YYYY-MM-DD-short-title.md
```

Example:

```text
2026-05-21-variational-autoencoders.md
```

Use this template:

```markdown
---
title: "Variational Autoencoders"
date: 2026-05-21
categories: study
tags:
  - Generative Models
  - PML
---

Write your note here.
```

## Publish on GitHub Pages

1. Create a GitHub repository named `your-github-username.github.io`.
2. Push these files to that repository.
3. In GitHub, open `Settings` -> `Pages`.
4. Set the source to your main branch and the root folder.
5. Visit `https://your-github-username.github.io`.

## Preview Locally

If Ruby 3.x and Bundler are installed:

```bash
bundle config set path vendor/bundle
bundle install
bundle exec jekyll serve
```

Then open `http://localhost:4000`.

The default macOS system Ruby may be too old for current GitHub Pages dependencies, so use a newer Ruby if you want local preview.
