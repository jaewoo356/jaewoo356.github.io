여태까지는 SRS 에서 뽑는 것이었다. 즉 다시 말해 뽑는 과정이 그저 i.i.d 하게 뽑는 것만 고려를 했었다. 하지만 이제는 i.i.d 하게 뽑기보다는 조금 다르게 뽑아줘서 추정량에 대한 분산을 줄여줘보겠습니다.

정확하게 그래서 어떤 얘기를 하느냐, 우선은 예시로 Xˉ\bar XXˉ 를 떠올려보겠습니다.

θ=E[X]\theta = \mathbb{E}[X]θ=E[X] 을 추정하기 위해서는 Xˉ\bar XXˉ 를 사용합니다. 이때 Xˉ\bar XXˉ 는 Unbiased + Consistent 합니다. 근데 MSE 를 가지고 Xˉ\bar XXˉ 가 얼마나 좋은지에 대한 지표도 구할 수가 있습니다.

그저 i.i.d 하게 뽑는 것만 고려한다면 샘플 개수인 nnn 을 올려서 추정량에 대한 분산, 즉 표준오차를 줄일 수가 있습니다. 하지만 시뮬레이션에서는 이것을 줄일 수 있는 여러 방법들이 존재합니다.

Antithetic Variables

예전에 분산을 줄이기 위해서 사용되었던 방법이며, Negative 하게 뽑아서 Negative Correlation 인 점을 활용합니다.

X1,X2X_1, X_2X1​,X2​ 를 i.i.d 하게 뽑아준다면 다음과 같이 나옵니다.

하지만 i.i.d 하기 때문에 당연하게도 공분산이 000 입니다. 그렇기에 분산은 사실 다음과 같습니다.

여기에서 분산을 그러면 어떻게 줄이냐, 바로 Cov(X1,X2)\mathrm{Cov}(X_1, X_2)Cov(X1​,X2​) 를 음수로 바꿔주는 것입니다. 가장 간단하게는 X1=h(U1,⋯ ,Um)X_1 = h(U_1, \cdots, U_m)X1​=h(U1​,⋯,Um​) 이면 X2=h(1−U1,1−U2,⋯ ,1−Um)X_2 = h\left(1-U_1, 1-U_2, \cdots, 1-U_m \right)X2​=h(1−U1​,1−U2​,⋯,1−Um​) 으로 바꿔주는 것입니다.

이렇게 진행한다면 Cov(X1,X2)<0\mathrm{Cov}(X_1, X_2) < 0Cov(X1​,X2​)<0 이기에 다음이 성립하게 됩니다.

간단한 예시로 생각해보겠습니다.

Example 1

θ=E[eU]\theta = \mathbb{E}\left[e^{U} \right]θ=E[eU] 를 추정하고 싶다고 하겠습니다. 우선은 분산, 공분산에 대한 값들을 다음과 같이 구하겠습니다.

일반적인 Monte-Carlo Estimate 방법에서는 그저 U1,U2,⋯ ,UnU_1, U_2, \cdots, U_nU1​,U2​,⋯,Un​ 까지 뽑은 다음에 이것을 eUie^{U_i}eUi​ 에 넣어준 다음에 1n∑i=1neUi\frac{1}{n}\sum_{i=1}^n e^{U_i}n1​∑i=1n​eUi​ 으로 평균을 내서 추정량을 만들어주겠죠.

222 개의 샘플에 대해서는 다음과 같이 분산을 구해주겠습니다.

이제는 Antithetic Variable 를 활용해보겠습니다.

똑같이 U1,(1−U1)U_1, (1-U_1)U1​,(1−U1​) 을 사용해보겠습니다. 그러면 eU1+e1−U12\frac{e^{U_1} + e^{1-U_1}}{2}2eU1​+e1−U1​​ 가 저희의 추정량이며 이것의 분산은 다음과 같습니다.

두 분산을 비교하면, 확실하게 이득을 본 모습입니다.

사실은 Monte-Carlo Estimate 에서 222 번 뽑을 것을 저희는 111 번만 뽑아서 사용하고 있습니다. 한 개로 두 개 효과를 보는 것이죠. 그렇기에 위 예시에서는 반만큼의 샘플들만 필요하다는 점도 있습니다.

이것에 대한 실제 코드와 함께 확인해보겠습니다.

rm(list=ls())
N = 10000
mc_estimator = exp(runif(N))
u_vals = runif(N / 2)
antithetic_estimator = c((exp(u_vals) + exp(1 - u_vals)) / 2)
cat("MC Estimator Mean : ", mean(mc_estimator), "Var : ", var(mc_estimator),
    "\nAntithetic Estimator : ", mean(antithetic_estimator), "Var : ", var(antithetic_estimator))

Example 2

원에서 π\piπ 구하는 예시로 돌아가보겠습니다. 이것을 저희는 이전에는 그저 (U1,U2)(U_1, U_2)(U1​,U2​) 들을 NNN 개 만큼 뽑은 다음에 1(U12+U22≤1)\mathbb{1}\left(U_1^2 + U_2^2 \le 1 \right)1(U12​+U22​≤1) 를 추정량으로 세웠습니다. 이게 Naive 한 MC Estimator 이지만, 이제는 다르게 (1−U1,1−U2)(1-U_1, 1-U_2)(1−U1​,1−U2​) 까지 활용해보겠으며, 코드로는 다음과 같습니다.

rm(list=ls())
# MC-Estimate
N = 10000
u1_vals = runif(N) 
u2_vals = runif(N)
circle_indicator = ifelse(u1_vals ** 2 + u2_vals ** 2 <= 1, 4, 0)
mc_est = mean(circle_indicator)
mc_var = var(circle_indicator)
mc_SE = sqrt(mc_var / N)

# Antithetic Estimator
M = N / 2
u1_vals = runif(M)
u2_vals = runif(M)
u1_u2_est = ifelse(u1_vals ** 2 + u2_vals ** 2 <= 1, 4, 0)
anti_u1_u2_est = ifelse((1 - u1_vals) ** 2 + (1 - u2_vals) ** 2 <= 1, 4, 0)

anti_circle_est = (u1_u2_est + anti_u1_u2_est) / 2

anti_est = mean(anti_circle_est)
anti_var = var(anti_circle_est)
anti_SE = sqrt(anti_var / M)

cat("MC Estimate :\n", "Mean : ", mc_est, "\tVar : ", mc_var, "\tSE : ", mc_SE,
    "\nAntithetic Estimate :\n", "Mean : ", anti_est, "\tVar : ", anti_var, "\tSE : ", anti_SE)

Control Variables

똑같이 θ=E[X]\theta = \mathbb{E}\left[X\right]θ=E[X] 을 추정하는게 목표지만, 이번에는 다음과 같은 YYY 라는 새로운 변수를 활용하겠습니다.

  • E[X+c(Y−μY)]=E[X]=θ\mathbb{E}\left[X + c\left(Y - \mu_Y\right) \right] = \mathbb{E}[X] = \thetaE[X+c(Y−μY​)]=E[X]=θ, Unbiased 하다.
  • Var(X+c(Y−μY))=Var(X+cY)=Var(X)+c2⋅Var(Y)+2c⋅Cov(X,Y)\mathrm{Var}\left(X + c\left(Y - \mu_Y\right) \right) = \mathrm{Var}\left(X + cY \right) = \mathrm{Var}(X) + c^2 \cdot \mathrm{Var}(Y) + 2 c \cdot \mathrm{Cov}(X, Y)Var(X+c(Y−μY​))=Var(X+cY)=Var(X)+c2⋅Var(Y)+2c⋅Cov(X,Y)

분산 부분을 살피면, 저희는 ccc 를 어떻게 고르냐에 따라서 추정량의 분산을 줄일지 올릴지 정해줄 수가 있습니다.

이것을 구하기 위해서는 ccc 에 대해서 미분을 시켜주면 됩니다.

ccc 를 위와 같이 대입시킨다면, 다음과 같이 분산 Term 이 바뀝니다.

Example 3

이전과 똑같은 예시를 가지겠습니다. θ=E[eU]\theta = \mathbb{E}\left[e^{U} \right]θ=E[eU] 를 Y=UY = UY=U 으로 세팅하겠습니다.

rm(list=ls())
N = 10000
u_vals = runif(N)
exp_u_vals = exp(u_vals)
c = - cov(u_vals, exp_u_vals) / var(u_vals)
mu_y = 0.5
control_estimators = exp_u_vals + c * (u_vals - mu_y)
mc_mean = mean(exp_u_vals)
mc_var = var(exp_u_vals)
mc_SE = sqrt(mc_var / N)
control_mean = mean(control_estimators)
control_var = var(control_estimators)
control_SE = sqrt(control_var / N)
cat("MC Estimators :\nMean :", mc_mean, "\tVar :", mc_var, "\tSE :", mc_SE,
    "\nControl Estimators : \nMean :", control_mean, "\tVar :", control_var, "\tSE :", control_SE)

Conditioning

EVVE 를 활용하면 다음과 같이 분산을 적어줄 수가 있습니다.

이 점을 활용해서 XXX 대신에 E[X∣Y]\mathbb{E}\left[X \mid Y\right]E[X∣Y] 으로 추정을 해주면 됩니다.

  • E[E[X∣Y]]=E[X]=θ\mathbb{E}\left[\mathbb{E}\left[X \mid Y \right] \right] = \mathbb{E}\left[ X \right] = \thetaE[E[X∣Y]]=E[X]=θ, Unbiased
  • Var(E[X∣Y])≤Var(X)\mathrm{Var}\left(\mathbb{E}[X \mid Y] \right) \le \mathrm{Var}(X)Var(E[X∣Y])≤Var(X)

그렇기에 저희는 사실은 추정량을 E[X∣Y]=g(Y)\mathbb{E}[X \mid Y] = g(Y)E[X∣Y]=g(Y) 으로 사용하고 있습니다.
