Random Permutation

가끔은 순서를 랜덤하게 바꾸어야 합니다. 예시로 저희는 Permutation y1,y2,⋯ ,yny_1, y_2, \cdots, y_ny1​,y2​,⋯,yn​ 이 있다고 하겠습니다. 저희의 목표는 이 순서를 완전히 랜덤하게 바꾸는 것이며, 이는 간단하게 이해시켜드리기 위해 y1,y2,y3,y4y_1, y_2, y_3, y_4y1​,y2​,y3​,y4​ 으로 설명해드리겠습니다.

그전에 설명을 하자면, xx x 는 색깔이 kkk 개인 위의 예시에서의 이산형 확률변수 (정확하게는 Categorical 변수) 라고 하겠습니다.

위를 Generalize 시키면, 다음과 같습니다.

  1. Permutation A=[y1,y2,⋯ ,yn]A = \left[y_1, y_2, \cdots, y_n\right]A=[y1​,y2​,⋯,yn​] 으로 시작하며 k=nk = nk=n 으로 세팅한다.
  1. U∼Unif(0,1)U \sim \mathrm{Unif}(0, 1)U∼Unif(0,1) 을 뽑고, I=⌊kU⌋+1I = \left\lfloor kU \right\rfloor + 1I=⌊kU⌋+1 을 구합니다.
  1. A[I]↔A[k]A[I] \leftrightarrow A[k]A[I]↔A[k] 즉 둘의 순서를 바꿉니다.
  1. k=k−1k = k-1k=k−1 으로 업데이트 시키며 2,32, 32,3 을 k=1k = 1k=1 이 도달할 때까지 반복합니다.

예시로 위의 Permutation 을 (1,2,3,4,5,6,7,8,9)(1, 2, 3, 4, 5, 6, 7,8,9)(1,2,3,4,5,6,7,8,9), 즉 n=9n = 9n=9 에서 진행하겠습니다. 그러면 Random Permutation 이 성공했다면 111-번째 자리에서 나올 값이 List 의 평균인 4.54.54.5 와 동일하겠죠. 이를 한번 코드로 실행해보겠습니다.

random_permutation <- function(x){
  n = length(x)
  for(k in n:2){
    u <- runif(1)
    I <- floor(k * u) + 1
    I_th <- x[I]
    x[I] <- x[k]
    x[k] <- I_th
    k = k - 1
  }
  return(x)
}


x <- c(1, 2, 3, 4, 5, 6, 7, 8, 9)
N <- 10000
x_1st_list <- rep(NA, N)
for(i in 1:N){
  x_1st_list[i] = random_permutation(x)[1]
}
cat("X 의 평균 : ", mean(x),
    "\nPermutation 의 평균 : ", mean(x_1st_list))

Acceptance-Rejection Method

우선은 Discrete 인 상황을 고려해보겠습니다. 예시로 아까 보았던 이산형 확률변수를 생각해보겠습니다.

위와 같은 알고리즘으로 진행하게 된다면 당연하게도 굉장히 비효율적입니다. for-loop 을 각 YYY 샘플에 대해서 사용해야하기 때문입니다. 심지어는 가장 높은 Threshold 까지 뽑아도 되지만 굳이 그 위까지 샘플링하고 있죠.
